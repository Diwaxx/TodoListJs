## Мини-приложение "Список задач"

### Цель

Сделать небольшое веб-приложение, которое:

- загружает список задач из API
- показывает задачи в виде списка или карточек
- позволяет фильтровать задачи
- открывает подробную информацию по выбранной задаче

Можно использовать:

- **React** - предпочтительно  
- или **чистый JavaScript**

Ориентир по времени выполнения: **около 1.5 часа**.

---

# 1. Данные (API)

Используй публичный API:

**Список задач**
https://jsonplaceholder.typicode.com/todos

**Список пользователей**
https://jsonplaceholder.typicode.com/users


---

# 2. Основной экран

При открытии страницы нужно:

- загрузить список задач
- вывести первые **15 задач**

Для каждой задачи показать:

- `title`
- `completed` - статус задачи
- `userId`

---

# 3. Фильтрация

Нужно реализовать:

### Поиск

Поиск по названию задачи (`title`).

### Фильтр по статусу

Сделать переключатель или select:

- все
- выполненные
- невыполненные

---

# 4. Детальная карточка задачи

При клике на задачу нужно открыть:

- отдельный блок
- модальное окно
- или отдельную страницу

В карточке показать:

- `title`
- статус (`completed`)
- `id`
- `userId`

Дополнительно нужно подтянуть имя пользователя по `userId` из API пользователей и показать:

- `name`
- `email`

---

# 5. Состояния

Нужно обработать:

- **loading** - пока данные загружаются
- **empty** - если по фильтру ничего не найдено
- **error** - если запрос не выполнился

---

# 6. UI требования

Минимальные, но обязательные:

- аккуратные отступы
- понятный список задач
- визуально заметный статус задачи
- кликабельность элементов
- адаптация под мобильный экран

Стилизовать можно как удобно:

- CSS
- Tailwind
- CSS modules

---

# 7. Технические требования

Нужно предоставить:

- **Git-репозиторий** или архив проекта
- **README**, в котором указано:
  - как запустить проект
  - какие технологии использованы
  - что бы доработал при наличии большего времени

Код должен быть:

- читаемым
- разбитым на понятные части
- без мусора и лишних закомментированных кусков

---

# 8. Что будет плюсом

Не обязательно, но можно сделать **1 пункт**, если останется время:

- сортировка задач по алфавиту
- debounce для поиска
- кнопка сброса фильтров
- отображение количества найденных задач

---

# Критерии оценки

## Обязательно

- приложение запускается
- задачи загружаются
- поиск работает
- фильтр по статусу работает
- открывается детальная карточка
- есть состояния **loading / empty / error**

## Плюсом будет

- аккуратный UI
- понятная структура компонентов
- нормальные названия переменных и файлов
- хороший README
